# 1.4 Variable assignment and Initialisation 

## Variable Assignment
* After a variable has been defined, you can give it a value (in a separate statement) using the ==*=*== operator. 
  * This process is called ==assignment== and the ==*=*== operator is called the **assignment operator**.

```
int width; // here we are defining an integer variable named width
width = 5; // assignment of value 5 into the variable width

//variable width now has value 5 
```
By default, assignment copies the value on the right-hand side of the = operator to the variable on the left-hand side of the operator. *This is known as **copy-assignment***

Once a variable has been given a value. the value of that variable can be printed via ==std::cout== and the ==<<== operator.

> Assignment can be used whenever we want to change the value held by a variable. Here's an example where we use assignment twice: 

```
#include <iostream>

int main() 
{
    int width ; // defining integer variable (width)
    width = 5; 

    std::cout << width; //prints 5 

    width = 7; // We have now changed the value stored in variable width to 7

    std::cout << width; //prints 7 

    return 0; 
}
```

This prints
```
>57
```

When the program above is run, execution begins at the top of the ==main()== function and proceeds sequentially: 
1. First, memory for the variable ==width== is allocated 
2. We then assign ==width== the value ==5==. 
3. When we output the value of ==width==, it prints ==5== to the console. 
4. When we then assign value ==7== to ==width==, any prior value (in this case ==5==) is overwritten.
5. Thus when we output ==width== again, this time it prints ==7==. 
   
> **Key point**
> Normal variables can only hold one value at a time. 

>**Warning**
==*=*== DOES NOT EQUAL == (More on this later)



## Variable Initialisation
The one downside of assignment is that assigning a value to a just-defined object requires two statements: one to define the variable and another to assign the value. 

>These two steps can be combined
>> When an object is defined, you can optionally provide an initial value for the object. 
>> This process of specifying an initial value for an object is called **initialisation** and the syntax used to initialise an object is called an initialiser. 

For example, the following statement both defines a variable named ==width== (of type ==int==) and initialises it with the value of 5: 

```
#include <iostream>

int main()
{
    int width = 5; // Intialisation of integer variable 'width' with initial value of 5
    std::cout << width; //prints 5 

    return 0;
}
```

In the above initialisation of variable ==width== , =={5}== is the initialiser and ==5== is the initial value. 



## Different forms of Initialisation
Unlike assignment (which is quite straightforward), in C++ initialisation has its own niches which are important to get correct: here is a simplified view to get started: 

There are 5 common forms of initialisation in C++: 

```
int a; // default-initialisation (no initialiser)

//Traditional Initialisation forms: 
int b = 5; // copy-initialisation (initial value after the equals sign)
int c (6); // direct-initialisation (initial value in parenthesis)

//Modern initialisation forms (preferred)
int d{7}; //direct-list-initialisation (initial value in braces)
int e{}; // value-initialisation (empty braces)
```

You may see the above forms written with different spacing ( e.g ==int b = 5;==, ==int d{7};==, ==int e{};==). Whether you use extra spaces for readability or not is a matter of personal preference. 

As of C++17, copy-initialisation (traditional), direct-initialisation (traditional) and direct-list-initialisation (modern) behave identically in most cases. 


## Default-Intialisation
When no initialiser is provided (such as for variable 'a' above), this is called default-initialisation. 
* In many cases, **default-initialisation** performs no initialisation and leaves the variable with an indeterminate value (a value that is not predictable, sometimes called a “*garbage value*”)

## Copy-Initialisation (Traditional)
When an initial value is provided after an equals sign, this is called **copy-initialisation.** 
This form of initialization was inherited from the **C language**.

```
int width = 5; // copy-intialisation of value 5 into the variable 'width'
```
>similar to copy-assignment, this copies the value on the right-hand side of the equals into the variable that is being created on the left-hand side

>**Usage**
Copy-initialisation had fallen out of favor in modern C++ due to being less efficient than other forms of initialisation for some complex types. 
However, C++17 remedied the bulk of these issues and copy-initialisation is now finding new advocates. 
You will also find it used in older code (especially code ported from C) or by developers who simply think it looks more natural and is easier to read.

## Direct Intialisation (Traditional)
When an initialisation is provided inside the parenthesis, this is called direct-intialisation. 
```
int width (5); //direct initialisation of value 5 into the variable with 'width'

```
>**Usage**:
Direct-initialisation was initially introduced to allow for more efficient initialisation of complex objects (those with class types, which we’ll cover in a future chapter). 
* Just like copy-initialisation, direct-initialisation had fallen out of favor in modern C++, largely due to being superseded by direct-list-initialisation. 
* However, direct-list-initialisation has a few quirks of its own, and so direct-initialisation is once again finding use in certain cases.
  
## List-initialisation (Modern)
The modern way to initialise objects in C++ is to use a form of initialisation that makes use of curly braces. 
>This is called **list-initialisation** (or **uniform initialisation** or **brace initialisation**).

List-intialisation comes in two forms: 
```
int width {5}; //direct-list intialisation of initial value 5 into variable width (preferred)
int height = {6}; //copy-list-intialisation of initial value 6 into variable height (rarely used)
```
>**informative tip:**
Prior to C++11, some types of initialisation required using copy-initialisation and other types of initialisation required using direct-initialization. 
>>Copy-initialisation can be hard to differentiate from copy-assignment (because both use an =) and direct-initialisation can be difficult to differentiate from function-related operations (because both use parentheses).

List-initialisation was introduced to provide a initialisation syntax that works in almost all cases, behaves consistently and has an unambiguous syntax that makes it easy to tell where we’re initialising an object.

>**Key Insight:**
When we see curly braces, we know we're list-initialising an object.

* Additionally, list-initialisation also provides a way to initialise objects with a list of values rather than a single value (which is why it is called “list-initialisation”). 
We show an example of this in lesson 16.2 - Introduction to std::vector and list constructors.

## List-intialisation dissalowing narrowing conversions
One of the primary benefits of list-initialistion for new Cpp programmers is that "narrowing conversions" are disallowed. 
This basically means that is try to list-initilialise a variable using a value that the variable cannot hold safetly (i.e using incorrect types)
